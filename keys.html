Database Keys

1. Primary Key

Definition: A unique identifier for a record in a table. Each table can have only one primary key, and it cannot contain NULL values.

StudentID Name Age Email
1 Alice 20 alice@example.com
2 Bob 22 bob@example.com
3 Charlie 23 charlie@example.com

Primary Key: StudentID

2. Foreign Key

Definition: A field (or a set of fields) in one table that uniquely identifies a row of another table. The foreign key is used to establish and enforce a link between the data in the two tables.

Students Table:

StudentID Name Age
1 Alice 20
2 Bob 22

Enrollments Table:

EnrollmentID StudentID CourseID
101 1 1001
102 2 1002

Foreign Key in Enrollments Table: StudentID (references StudentID in Students Table)

3. Composite Key

Definition: A key that consists of two or more columns to uniquely identify a record in a table.

OrderID ProductID Quantity
1 100 2
1 101 1
2 100 1

Composite Key: (OrderID, ProductID)

4. Candidate Key

Definition: A field or combination of fields that can qualify as a unique key in a table. There can be multiple candidate keys in a table.

EmployeeID SSN Email
1 123-45-6789 alice@example.com
2 987-65-4321 bob@example.com

Candidate Keys: EmployeeID, SSN

5. Alternate Key

Definition: A candidate key that is not chosen as the primary key.

EmployeeID SSN Email
1 123-45-6789 alice@example.com
2 987-65-4321 bob@example.com

Primary Key: EmployeeID

Alternate Key: SSN

6. Artificial Key

Definition: A key created solely for the purpose of uniquely identifying records in a table, often when no natural primary key exists. Also known as a surrogate key.

CustomerID CustomerNumber Name Email
1 C001 Alice alice@example.com
2 C002 Bob bob@example.com

Artificial Key: CustomerID

These keys help in maintaining data integrity, establishing relationships, and ensuring the uniqueness of records within a database.
